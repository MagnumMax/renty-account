# AI Coding Rules (Vibecoding Standard)

**Система тегов:**
- `[UI/UX]` — Фокус на дизайн, верстку, адаптивность и плавность переходов (Vanilla CSS/Tailwind, семантический HTML).
- `[DATA]` — Работа с мок-данными (JSON), структура локального стейта, проектирование будущих API-запросов.
- `[CORE]` — Бизнес-логика, маршрутизация (SPA роутинг на JS), обработка событий оплаты и продления.

**Правила написания:**
- **Data Before UI:** Сначала проектируем структуру мок-данных (`mocks.js` / JSON), потом натягиваем на нее UI.
- **Атомарные коммиты и шаги:** Не пытаться реализовать весь кабинет за один шаг. Разделять верстку Layout, Dashboard, Модалок и Настроек.
- **Чистый код (Vanilla):** Пишем современный, но чистый HTML5, CSS3 (с переменными) и ES6+ JavaScript. Избегаем ненужных зависимостей и тяжелых библиотек для MVP.
- **Семантика и Доступность:** Использовать правильные теги (`<nav>`, `<main>`, `<article>`), `data-` атрибуты для стейтов, обеспечивая базовую доступность.
