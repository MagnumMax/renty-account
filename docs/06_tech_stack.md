# 🛠 Tech Stack

Так как сейчас стоит задача сделать максимально быструю и простую интеграцию в существующий сайт без усложнения инфраструктуры, мы отходим от тяжеловесного стандарта Next.js и выбираем классический, легковесный стек для создания SPA (Single Page Application), который можно просто положить в папку на текущем сервере.

**Frontend:**
- **Архитектура:** Vanilla HTML / SPA (Single Page Application)
- **Стилизация:** Чистый CSS с использованием CSS Variables (Custom Properties), определенных в `DESIGN.md`. Для сложных компонентов и сетки можно использовать легковесный фреймворк (например, Tailwind CSS через CDN или локальную сборку, либо Bootstrap 5, если он уже используется на сайте). *Рекомендуется: Tailwind CSS через CDN для скорости разработки MVP.*
- **JavaScript:** Vanilla JavaScript (ES6+ Modules) или легковесная библиотека вроде Alpine.js / Vue 3 (через CDN) для реактивности, если потребуется сложная логика модалок и переключений табов. *Рекомендуется: Vanilla JS для максимальной независимости.*
- **Иконки:** Ionicon или FontAwesome (через CDN, если они уже есть на сайте).
- **Шрифты:** Наследуются от основного сайта (Google Fonts: Ubuntu, Changa One).

**Backend, Auth & Database (Мок-уровень для MVP):**
- **Data Layer:** Локальные JSON-файлы (Mock Data) или хардкод в JS для симуляции ответа API.
- **State Management:** `localStorage` или `sessionStorage` для сохранения состояния (например, "Авторизован", "Оплачено") между перезагрузками страницы во время тестирования прототипа.

**Backend, Auth & Database (Для Phase 1 - Реальная интеграция):**
- **Auth Provider:** Рекомендуется использовать готовые BaaS решения (например, **Supabase Auth** или **Firebase Auth**), так как они "из коробки" предоставляют отправку OTP-кодов на почту, интеграцию с Google/Apple Sign-In и поддержку Passkeys, избавляя от необходимости писать этот сложный и секьюрный код с нуля.

**Инфраструктура:**
- Не требует отдельного Node.js сервера.
- Может быть отдан любым статичным сервером (Nginx, Apache) вместе с основным сайтом.

**Актуальные Best Practices для Vanilla SPA (2026):**
- Использовать `data-` атрибуты в HTML для управления состоянием UI (например, `data-state="active"`).
- Разделять JS логику на модули (файлы), даже без сборщика, используя `<script type="module">`.
- Использовать CSS-переменные для легкой поддержки светлой/темной темы через `prefers-color-scheme` и переключатель классов на `<body>`.
