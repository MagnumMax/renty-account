# 🎨 Renty Client Cabinet – Design Kit & Tokens
*Стиль: Seamless Native (Строгое наследование основного сайта renty.com)*

Этот документ содержит дизайн-токены и правила UI для интеграции кабинета клиента в существующий сайт. Кабинет должен ощущаться как органичное продолжение основного продукта.

## 1. 🌈 Цветовая палитра (Color Tokens)

Кабинет использует только светлую тему, чтобы интерфейс оставался максимально консистентным с основным сайтом.

### Основные цвета (Brand & Accent)
- **`--primary`**: `hsl(211, 100%, 35%)` /* Фирменный синий/акцентный цвет Renty */
- **`--primary-hover`**: `hsl(211, 100%, 30%)`
- **`--secondary`**: `hsl(219, 14%, 60%)` /* Нейтральный акцент для второстепенных действий */
- **`--accent`**: `hsl(35, 100%, 50%)` /* Оранжевый/Желтый для бейджей "Premium", "Upsell" */

### Фоны (Backgrounds & Surfaces)
- **`--bg-main`**: `hsl(0, 0%, 98%)` /* Чуть серый фон для контраста с карточками */
- **`--bg-surface`**: `hsl(0, 0%, 100%)` /* Белые карточки */
- **`--bg-surface-hover`**: `hsl(216, 38%, 95%)`

### Текст (Typography Colors)
- **`--text-primary`**: `hsl(210, 11%, 15%)`
- **`--text-secondary`**: `hsl(219, 14%, 40%)`
- **`--text-disabled`**: `hsl(219, 14%, 80%)`

### Статусы (State/Semantic Colors)
- **`--success`**: `hsl(152, 63%, 43%)` /* Успешная оплата, активная бронь */
- **`--warning`**: `hsl(35, 100%, 50%)` /* Ожидание оплаты, истекающая бронь */
- **`--error`**: `hsl(0, 79%, 63%)` /* Ошибка оплаты, отмена */

## 2. ✍️ Типографика (Typography)

Наследование шрифтов основного сайта для бесшовного перехода.

- **Основной шрифт (Sans-serif)**: `'Ubuntu', 'Inter', sans-serif`
- **Заголовки (Headings)**: `'Changa One', 'Ubuntu', sans-serif`

**Размеры (Scale):**
- `text-xs`: 0.75rem (12px) - Бейджи, мелкие подписи
- `text-sm`: 0.875rem (14px) - Второстепенный текст, лейблы инпутов
- `text-base`: 1rem (16px) - Основной текст
- `text-lg`: 1.125rem (18px) - Заголовки карточек машин
- `text-xl`: 1.25rem (20px) - Заголовки секций
- `text-2xl`: 1.5rem (24px) - Главный заголовок дашборда

## 3. 📐 Формы и Отступы (Geometry & Spacing)

Строгий, утилитарный, но современный внешний вид.

### Скругления (Border Radius)
- **`--radius-sm`**: `4px` /* Чекбоксы, бейджи */
- **`--radius-md`**: `8px` /* Кнопки, инпуты */
- **`--radius-lg`**: `12px` /* Карточки бронирований, модалки */
- **`--radius-xl`**: `16px` /* Крупные виджеты дашборда */

### Тени (Shadows)
- **`--shadow-sm`**: `0 1px 2px rgba(0,0,0,0.05)` /* Инпуты, кнопки */
- **`--shadow-md`**: `0 4px 6px -1px rgba(0,0,0,0.1)` /* Карточки */
- **`--shadow-lg`**: `0 10px 15px -3px rgba(0,0,0,0.1)` /* Дропдауны, модалки */

---

## 4. 🧩 UI-Паттерны для Кабинета (Vibe Elements)

1. **Native Integration:** Кабинет не должен выглядеть как SPA-приложение, оторванное от сайта. Он использует тот же header/footer, что и renty.com.
2. **Booking Cards (Карточки бронирования):** Главный элемент интерфейса. Содержит фото автомобиля, даты, статус (Активно/Завершено) и быстрые действия (Продлить, Оплатить).
3. **Data Density:** Интерфейс должен быть плотным, чтобы пользователь мог быстро увидеть детали платежа, инвойсы и доп. услуги (Upsells) без лишнего скролла.
4. **Interactive States:** Четкая визуальная обратная связь при наведении (hover) на карточки и кнопки (изменение `--bg-surface` на `--bg-surface-hover`).
