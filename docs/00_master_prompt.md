# 🧠 Master Prompt (Vibecoding Context)
*Этот файл объединяет выжимку из всей документации. Передай его ИИ-агенту при создании нового чата.*

**Product:**
Кабинет клиента (B2C) для маркетплейса Renty. Главные функции: управление активными бронями, продление аренды, покупка дополнений (Upsells), оплата инвойсов, а главное — управление штрафами (с пруф-фотографиями) и доп. расходами с возможностью оплаты по Payment Link. Предусмотрен Soft Delete аккаунта.

**Vibe:**
Стиль "Seamless Native". Строгое наследование дизайна renty.com. Интерфейс должен восприниматься как органичная часть существующего сайта, а не отдельное приложение. Плотная верстка, акцент на карточки бронирований. Основные цвета: синий бренд-цвет `hsl(211, 100%, 35%)`, поддержка светлой и темной темы. Шрифты: Ubuntu и Changa One.

**Core Features:**
- Авторизация: Passwordless (Email OTP, Google/Apple Sign-In, Passkeys).
- Dashboard с быстрым доступом к активной брони и алертами.
- Роутинг: `/dashboard`, `/bookings` (с модалками продления/отмены/дополнений), `/invoices` (с табами для штрафов), `/wishlist`, `/settings` (профиль и soft delete).
- Навигация: Левый сайдбар (десктоп) и гамбургер-шторка (мобилки).

**Data & Stack:**
- **Stack:** Легковесное HTML/CSS/JS SPA. Без Node.js, без тяжелых фреймворков. Максимально простая интеграция в существующий сервер.
- **Data (MVP):** Локальные моки внутри HTML/JS.
- **Data Architecture (Target):** Сущности User (с `is_active` для soft delete), Booking, BookingAddon (для upsells), Invoice, Fine (отдельная таблица для штрафов с `evidence_urls`).
