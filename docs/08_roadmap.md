# 🛣 Roadmap & Phasing

## 🎯 MVP (Minimum Viable Product - Frontend HTML Prototype)
**Фокус:** Создание статического, но интерактивного HTML/JS прототипа. Кликабельный интерфейс, UI/UX, навигация.
**СТРОГОЕ ПРАВИЛО:** На этом этапе ЗАПРЕЩЕНО подключать реальную БД, сервер или Auth. Используем исключительно захардкоженные HTML/JSON данные и Vanilla JS для имитации поведения.

- [ ] Создать базовую HTML-структуру (Layout) с левым сайдбаром (десктоп) и гамбургером (мобилки).
- [ ] Сверстать `Dashboard` с карточкой активного бронирования и алертом о штрафе.
- [ ] Сверстать страницу `Bookings` со списком заказов.
- [ ] Сверстать модальные окна/состояния для "Продлить аренду" и "Управление дополнениями".
- [ ] Сверстать страницу `Invoices & Fines` с карточками штрафов (с фото).
- [ ] Написать Vanilla JS логику для переключения "страниц" (табов) без перезагрузки, чтобы имитировать SPA.

## 🚀 Phase 1 (API Integration)
**Фокус:** Подключение статического HTML к реальному Backend API маркетплейса Renty.
- [ ] Интеграция с существующей системой авторизации Renty.
- [ ] Замена статических JSON моков на AJAX (fetch) запросы к реальному REST API.
- [ ] Интеграция Stripe Payment Links для кнопок "Оплатить".

## 🌟 Phase 2 (Scale & Advanced Features)
**Фокус:** Улучшение пользовательского опыта и расширение функционала.
- [ ] Полноценный Soft Delete профиля с отправкой email-уведомлений.
- [ ] Скачивание динамически генерируемых PDF-инвойсов.
- [ ] Мультиязычность интерфейса кабинета.
